Basic Structure

An HTML doc is structured using set of nested tags, each enclosed in angle brackets and acting as container for content or other tags

  • HTML is structured using nested tags
  • Each tag enclosed in angle brackets
  • < !DOCTYPE html > specificies document type
  • < html > root element tag encapsulates all content
  • Can include language (eg < html lang="en" >)
  • < head > tag contains metadata and links to external CSS + JS
  • < title > tag sets title of web page (displayed on tab bar)
  • < body > tag contains visible content of web page (text, images, etc)

HTML Comments

Comments are ignored by web browsers and aid in readability / documentation. They are denoted by < !-- content -- > and can be single or multiple lines. Ctrl (Cmd) + / is the shorthand. See exmples in code (they won't show up below).


HTML Tags

Tags are words contained in angle brackets. A small set of tags make up 70+% of HTML content

  • Doc Structure Tags
    • < !DOCTYPE html >: Specifies document type
    • < html >: Encloses entire HTML doc
    • < head >: Contains meta-information and links to CSS + JS
    • < body >: Contains main content of webpage
  • Metadata Tags
    • < title >: Sets title of web page
    • < meta >: Provides metadata such as charset, author, viewport settings
    • < link >: Links to external resources like CSS
  • Text Formatting Tags
    • < p >: Paragraph
    • < h1 >: For headings (sizes h1-h6)
    • < pre >: Preformatted text useful for displaying code snippets
    • < strong >: Strong emphasis (usually bold)
    • < em >: Emphasis (typically italic)
    • < br >: Line break
    • < hr >: Horizontal rule
  • List Tags
    • < ul >: Unordered list
    • < o1 >: Ordered list
    • < li >: List item
  • Hyperlink and Media Tags
    • < a >: Anchor (used for links)
    • < img >: Image content
    • < audio >: Audio content
    • < video >: Video content
  • Form Tags
    • < form >: Form
    • < img >: Input field
    • < textarea >: Text area
    • < button >: Button
    • < select >: Dropdown list
    • < option >: Option with a select or datalist
  • Table Tags
    • < table >: Table
    • < tr >: Table row
    • < td >: Table data cell
    • < th >: Table header cell
    • < thead >: Table header group
    • < tbody >: Table body group
    • < tfoot >: Table footer group
  • Semantic Tags
    • < header >: Header section
    • < footer >: Footer section
    • < article >: Article
    • < section >: Section
    • < nav >: Navigation
    • < aside >: Sidebar content

Paired tags come with opening and closing and contain content (close has /). Common examples include p, b, and headers (h1-h6).

Unpaired tags don't require closing tag - they're self-contained. Common examples include br, hr, and img.


HTML Elements

An HTML element consists of a start tag, content, and end tag. See example below:
< h1 >This is our first heading< /h1 >

Elements can also be nested (can be parent while tags cannot). See example below:
< h1 >< b >This is our first heading< /b >< /h1 >

Elements can also be empty (eg the break tag). Self closing tags are still considered elements.


HTML Attributes

HTML attributes are used to defined characteristics of an HTML element. They are placed in the element's opening tag and include a name (propert) and value. There are three main types:

  • Core attributes: Basic attributes that can be applied to most elements including id, class, and style
  • Internationalization attributes: Attributes helping adapt to specific locales (eg lang, dir)
  • Generic attributes: Attributes that provide additional

Core attributes are the most widely used. See examples below:

  • id: Unique identifier, can only be used on single element, often used with JS
    • < h4 id="myUniqueId" >Some text< h4 >
  • class: Can be applied multiple times, usually camelcase (eg myClass)
    • < h4 class="myClass" >Some textt< h4 >
  • title: Provide more info on hover
    • < h4 class="hello" >Some text< h4 >
  • style: Used directly inline for styling individual elements

CSS can also be applied in the style section (or a style sheet). IDs are referenced with #, while classes can be dot referenced: 

    /* CSS for ID */
    #myUniqueID { 
        background-color: yellow;
    }
    /* CSS for class */
    .myClass {
        font-size: 18px;
    }

Headings Tag

Headings from h1 to h6 are used to hierarchically structure content on web pages. This is helpful for both users and search engines (eg Google)

  • h1: Often used for main heading or title
  • h2: Often used for second-level headings, or section titles
  • h3: Often used for subsections in h2 sections
  • h4: Often used for subsections in h3 sections
  • h5: Rarely used but sometimes helpful for deeply nested sections
  • h6: Rarely used but sometimes helpful for specific formatting needs

Horizontal Rule & Line Break Tags

The hr and br tags are both self closing tags. The horizontal rule helps visually separate different sections of your doc with a horizontal line, while the br breaks lines of text (especially helpful for addresses, poems, lyrics, etc where preserving original line structure is important).


Anchor Tag

Anchor tags (< a >) are used to create links. They're also known as hyperlinks and used to link one doc to another. All anchor tags must have the href attribute, defining what URL the link points to. The target attribute is optional and defines where to open the linked document. See target options below:

  • _self: Opens document in same window or tab (default)
  • _blank: Opens document in new window or tab
  • _top: Opens document in full body of the window
  • _parent: Opens document in parent frame
Google Homepage

Links typically appear in different colors based on their state (customizable with CSS):

  • Active: Displayed in red and underlined
  • Visited: Displayed in purple and underlined
  • Unvisited: Displayed in blue and underlined

Anchor tags can also be used to link to sepecific sections in webpages. You can either use the name or id attribute of the target section, or use a hyperlink with a # followed by the target id or name (eg < a href="#section-one">Go to Section 1< /a >).

Image Tag

Image tags are used to embed images into websites. The basic syntax looks like this: < img src="image's path" />. Images are self-closing tags, and alt is often used to give an image description. PNG, JPEG, JPG, and GIF are all supported. Image dimensions can also be set via width and height, though CSS is best practice (exception: specifying image dimensions can sometimes impact SEO and improve Cumulative Layout Shift score.
